# models/

ML interatomic potential weights (ONNX format, e.g. ANI-2x / MACE / Allegro exports
consumed by `onnxruntime-web` on the WebGPU backend) live in this directory.

These files are deliberately **not** covered by a blanket `.gitignore` rule -- see
the root `.gitignore` note. Large `*.onnx` artifacts should be committed here
consciously (or, once the repo grows, moved to Git LFS or attached as GitHub
release assets -- that decision is out of scope for the initial scaffold).

Naming convention (proposed, adjust as real models land):

```
models/
  <family>-<variant>.onnx        e.g. ani2x-b973c.onnx
  <family>-<variant>.json        sidecar metadata: element order, cutoff radius,
                                  training set citation, output units
```

## `ani2x/` — ANI-2x weights (pure-TS format)

Populated in wave two. The engine's ANI-2x provider does **not** use ONNX: the
AEV featurizer and per-element MLPs run in pure float64 TypeScript (see
`packages/engine/src/potentials/ani2x/`), so the weights ship as a compact binary
blob rather than an `.onnx` graph:

```
ani2x/
  manifest.json               AEV params, self energies, per-element layer dims,
                              celu_alpha, and the variant list
  weights-full-f32.bin        8 ensemble members, float32   (52.3 MiB)
  weights-single-f32.bin      member 0 only, float32        (6.5 MiB)
  weights-full-f16.bin        8 members, IEEE half          (26.1 MiB)
  weights-single-f16.bin      member 0 only, half           (3.3 MiB)
```

Blob layout: `for member: for element(H,C,N,O,S,F,Cl): for layer: W[out*in]
row-major, then b[out]`. Generated by `spike/ani2x-onnx/export_weights.py`.
`full-f32` is effectively exact vs TorchANI (ANI-2x ships as float32; evaluating
those exact bits in float64 reproduces the double-precision reference). Load via
`Ani2xProvider.create({ modelDir, variant })`.
